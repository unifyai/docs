Convert angular pixel co-ordinates image in frame 1 \(\mathbf{A}_{p1}\in\mathbb{R}^{is×3}\) to angular pixel co-ordinates image in frame 2 \(\mathbf{A}_{p2}\in\mathbb{R}^{is×3}\).

[reference]

Parameters:

  • angular_pixel_coords1 – Angular pixel co-ordinates image in frame 1 [batch_shape,image_shape,3]

  • cam1to2_ext_mat – Camera1-to-camera2 extrinsic projection matrix [batch_shape,3,4]

  • pixels_per_degree – Number of pixels per angular degree

  • batch_shape – Shape of batch. Inferred from inputs if None. (Default value = None)

  • image_shape – Image dimensions. Inferred from inputs in None. (Default value = None)

ivy_vision.two_view_geometry.closest_mutual_points_along_two_skew_rays(camera_centers, world_ray_vectors, batch_shape=None, image_shape=None, dev_str=None)[source]#

Compute closest mutual homogeneous co-ordinates \(\mathbf{x}_{1,i,j}\in\mathbb{R}^{4}\) and \(\mathbf{x}_{2,i,j}\in\mathbb{R}^{4}\) along two world-centric rays \(\overset{\sim}{\mathbf{C}_1} + λ_1\mathbf{rv}_{1,i,j}\) and \(\overset{\sim}{\mathbf{C}_2} + λ_2\mathbf{rv}_{2,i,j}\), for each index aligned pixel between two world-centric ray vector images \(\mathbf{RV}_1\in\mathbb{R}^{is×3}\) and \(\mathbf{RV}_2\in\mathbb{R}^{is×3}\). The function returns two images of closest mutual homogeneous co-ordinates \(\mathbf{X}_1\in\mathbb{R}^{is×4}\) and \(\mathbf{X}_2\in\mathbb{R}^{is×4}\), concatenated together into a single array.

[reference] second answer in forum

Parameters:

  • camera_centers – Camera center [batch_shape,2,3,1]

  • world_ray_vectors – World ray vectors [batch_shape,2,image_shape,3]

  • batch_shape – Shape of batch. Inferred from inputs if None. (Default value = None)

  • image_shape – Image dimensions. Inferred from inputs in None. (Default value = None)

  • dev_str – device on which to create the array ‘cuda:0’, ‘cuda:1’, ‘cpu’ etc. Same as x if None. (Default value = None)

Returns:

ret – Closest mutual points image [batch_shape,2,image_shape,4]

Compute fundamental matrix \(\mathbf{F}\in\mathbb{R}^{3×3}\) between two cameras, given their extrinsic matrices \(\mathbf{E}_1\in\mathbb{R}^{3×4}\) and \(\mathbf{E}_2\in\mathbb{R}^{3×4}\).

[reference] bottom of page 244, section 9.2.2, equation 9.1

Parameters:

  • full_mat1 – Frame 1 full projection matrix [batch_shape,3,4]

  • full_mat2 – Frame 2 full projection matrix [batch_shape,3,4]

  • camera_center1 – Frame 1 camera center, inferred from full_mat1 if None [batch_shape,3,1] (Default value = None)

  • pinv_full_mat1 – Frame 1 full projection matrix pseudo-inverse, inferred from full_mat1 if None [batch_shape,4,3] (Default value = None)

  • batch_shape – Shape of batch. Inferred from inputs if None. (Default value = None)

  • dev_str – device on which to create the array ‘cuda:0’, ‘cuda:1’, ‘cpu’ etc. Same as x if None. (Default value = None)

Returns:

ret – Fundamental matrix connecting frames 1 and 2 [batch_shape,3,3]

Triangulate depth in frame 1, returning depth scaled homogeneous pixel co-ordinate image \(\mathbf{X}\in\mathbb{R}^{is×3}\) in frame 1.

  • image_shape – Image dimensions. Inferred from inputs in None. (Default value = None)
